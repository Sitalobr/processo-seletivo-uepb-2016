Esta avaliação consiste em:

 

  1. Compreender a estrutura deste projeto MVC, cujos principais pacotes de classes estão divididos nos seguintes pacotes: dao, model e beans. 
  2. Completar as funcionalidades abaixo, considerando que as classes contidas nos pacotes dao e model estão completas e que não há necessidade de alterações nessas classes. 

 


a) Completar o Cadastro de Disciplinas de um Curso(disciplinas.xhtml);
b) Cadastrar Alunos. Observe o modelo do Aluno (Aluno.java) na construção da view (cadastrarAlunos.xhtml).
c) Matricular Alunos em Disciplinas. Esta tarefa consiste em associar alunos as disciplinas, em um determinado período. Implemente da maneira que achar mais conveniente (matricularEmDisciplina.xhtml);
d) Construcao de um Web Service. Esta funcionalidade consiste em um servico REST que retornar o numero de alunos por extenso. Ex.: getNumExtenso(4) return "quatro".
e) Informar Notas dos Alunos. Esta funcionalidade consiste em escolher uma disciplina de um determinado período e possibilitar a digitação de quaisquer notas dos alunos matriculados na disciplina (digitacaoDeNotas.xhtml).


obs:

  1. Classes do pacote "bean" podem ser criadas/alteradas conforme a necessidade.
  2. os dados persistidos estão sendo armazenados em memória. Portanto, dados persistidos existiram apenas enquanto o servidor de aplicação estiver em execução, juntamente com a aplicação. 
  3. Observe que na interface de listagem de curso, há um botão que possibilita a inserção (persistência) e listagem de alguns cursos para testes.
  4. o projeto utiliza CDI para injenção de dependências, não havendo necessidade de instanciar classes do pacto DAO(Observe como exemplo a clase CursoBean.java).
  5. A utilização de componentes do PrimeFaces é opcional.
